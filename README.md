# Quick Reference Guides: Specs, Skills & Tools

Printable one-page references for AI engineers who need a fast, structured reminder of what to include — and how to organize it — when building specifications, skills, and tools. These guides distill the key patterns without requiring a full read of the underlying documentation.

---

## Who These Are For

AI engineers and developers building agent systems who want:

- A reliable checklist when authoring specs, skills, or tools
- A consistent structure they can return to without re-reading lengthy docs
- A printable reference to keep close during active development

New to the concepts? These guides are designed to be approachable. Each QRG presents the structure clearly so you can understand the "what" and "why" at a glance.

---

## Platform Compatibility

These guides are **optimized for the Anthropic ecosystem** (Claude Code, Claude API, Claude Desktop, Claude.ai), but the core principles — progressive disclosure, metadata-driven discovery, structured decision criteria — are **model-agnostic** and apply across platforms.

> **Note on Skills & Google Gemini:** Gemini loads skills differently than Anthropic. Gemini uses **system instructions** for skills rather than a dedicated skill-loading mechanism.

For full cross-platform details including OpenAI, Google (Vertex AI / AI Studio), and open-source models, see the companion resource:

**[Appendix D: Cross-Platform Implementation & Resources](https://archiecur.github.io/ai-system-design/skills/Skills_D_Cross-Platform_Implementation_Resources_Appendix/)**  
Covers platform-specific implementations, terminology mapping, conversion tips, and links to official documentation for each platform.
